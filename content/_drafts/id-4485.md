## memo()

This is used to create a pure function that will only be re-rendered if it’s state or props change (i.e., data that it actually depends on)

Example:

```javascript
function Decoration() {
  return (
    <div className="decoration">
      ⛵️
    </div>
  );
}

const PureDecoration = React.memo(Decoration);

export default PureDecoration;
```

## useMemo()

This can be used for two different purposes.

First, it is used to preserve / “remember” a computed value between re-renders - think heavy computations.

Example:

```javascript
/* Without memoization */
const allPrimes = [];
  for (let counter = 2; counter < selectedNum; counter++) {
    if (isPrime(counter)) {
      allPrimes.push(counter);
    }
  }

/* With memoization */
const allPrimes = React.useMemo(() => {
  const result = [];

  for (let counter = 2; counter < selectedNum; counter++) {
    if (isPrime(counter)) {
      result.push(counter);
    }
  }

  return result;
}, [selectedNum]);
```

Second, it can be used to preserve references to actual instances of arrays (since arrays are compared by reference, not actual value). If a component re-renders, even if the data within an array being passed is the same, it is still a new array that has been created (!==).

```javascript
function getNumbers() {
  return [1, 2, 3];
}

const firstResult = getNumbers();
const secondResult = getNumbers();

console.log(firstResult === secondResult);
```

Example:

```javascript
/* Without memoization */
// Every time we render this component, we call this function...
function App() {
  // ...and wind up creating a brand new array...
  const boxes = [
    { flex: boxWidth, background: 'hsl(345deg 100% 50%)' },
    { flex: 3, background: 'hsl(260deg 100% 40%)' },
    { flex: 1, background: 'hsl(50deg 100% 60%)' },
  ];

  // ...which is then passed as a prop to this component!
  return (
    <PureBoxes boxes={boxes} />
  );
}
```

```javascript
/* Refactor the boxes array to use memoization */
const boxes = React.useMemo(() => {
  return [
    { flex: boxWidth, background: 'hsl(345deg 100% 50%)' },
    { flex: 3, background: 'hsl(260deg 100% 40%)' },
    { flex: 1, background: 'hsl(50deg 100% 60%)' },
  ];
}, [boxWidth]);
```

## useCallback()

The `useCallback` hook is the same as `useMemo` but it is for remembering/preserving function references, as opposed to arrays.

```javascript
const functionOne = function(){ return 5; };
const functionTwo = function(){ return 5; };

console.log(functionOne === functionTwo); // false
```

Example:

```javascript
/* Full component without memoization */
function App() {
  const [count, setCount] = React.useState(0);

  function handleMegaBoost() {
    setCount((currentValue) => currentValue + 1234);
  }

  return (
      <MegaBoost handleClick={handleMegaBoost} />
  );
}
```

```javascript
/* Refactor handleMegaBoost with memoization via useCallback */
const handleMegaBoost = React.useCallback(() => {
  setCount((currentValue) => currentValue + 1234);
}, []);
```

We could alternatively still use `useMemo` and return a function, as follows:

```javascript
/* Refactor handleMegaBoost with memoization via useMemo */
const handleMegaBoost = React.useMemo(() => {
  return function() {
    setCount((currentValue) => currentValue + 1234);
  }
}, []);
```

This last approach is not really conventional. Essentially, `useCallback` is “syntactic sugar” to make our lives easier. So the two following expressions actually have the same effect:

```javascript
// This:
React.useCallback(function helloWorld(){}, []);

// ...Is functionally equivalent to this:
React.useMemo(() => function helloWorld(){}, []);
```
